## Technical Discoveries

### GGUF v3 Metadata Format (critical fix)
```
metadata_entry:
  key: string (8-byte len + data)
  value_type: uint32
  value: variable (depends on type)
```
My original code read `value_type` as the first field and interpreted `key_len` bytes as value_type. Caused "key_len" of 7308890738324930560 and 0x14=20 as invalid type.

### CMA Depletion Root Cause
After the first ggml_init() call, the GPU driver allocated CMA for CUDA and never released it. From 344MB вҶ’ ~20MB free. Now at 644KB. Only fix is reboot.

### ggml Pip Package CUDA Binding
The llama-cpp-python pip package compiles ggml with CUDA. Even `libggml.so` (the tiny backend loader) links to `libggml-cuda.so` which links CUDA libs. Can't extract a CUDA-free subset.

### Key Decision
- **Threaded streaming is the right approach for Evennia** вҖ” Evennia's Twisted reactor is single-threaded. Blocking inference in the command handler means messages queue but don't flush until command returns. Spawning a background thread and using `reactor.callFromThread` for `caller.msg()` lets tokens appear progressively. The C mutex serializes concurrent access when multiple users call @infer.
